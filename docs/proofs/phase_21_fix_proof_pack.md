# Phase 21 Fix Proof Pack

`eve fix` is a safe facade over the governed repair pipeline. It inspects a target repository, selects one actionable issue, proposes a minimal fix, writes target-local evidence, and only mutates files with `--apply`.

## Command Format

```bash
cargo run -- fix <TARGET_PATH> [--only ci|tests|docs|cargo-check] [--apply] [--no-llm]
```

## Proof Cases

| Case | Setup | Command | Expected Behavior | Observed Result |
| --- | --- | --- | --- | --- |
| `missing_ci` | Rust lib project without `.github/workflows/rust-ci.yml` | `cargo run -- fix "$TARGET" --only ci --apply --no-llm` | detects `missing_ci`, creates `.github/workflows/rust-ci.yml`, reports `validation_passed`, writes evidence under `$TARGET/.eva/fix/<fix_id>/`, may create `Cargo.lock` as validation side effect | generated by proof helper or test fixture |
| `missing_smoke_test` | Rust lib project without `tests/eve_smoke.rs` | `cargo run -- fix "$TARGET" --only tests --apply --no-llm` | detects `missing_smoke_test`, creates `tests/eve_smoke.rs`, records validation honestly, writes target-local evidence | generated by proof helper or test fixture |
| `readme_missing_validation` | Rust lib project with `README.md` missing validation section | `cargo run -- fix "$TARGET" --only docs --apply --no-llm` | detects missing README validation section, updates `README.md`, reports source mutation, writes target-local evidence | generated by proof helper or test fixture |
| `simple_missing_module` | Rust crate with `mod missing_module;` and missing `src/missing_module.rs` | `cargo run -- fix "$TARGET" --only cargo-check --apply --no-llm` | detects cargo-check failure, creates `src/missing_module.rs`, validation passes or is recorded honestly, no semantic overclaim | generated by proof helper or test fixture |
| `unknown_empty_project` | Empty directory without `Cargo.toml` | `cargo run -- fix "$TARGET" --apply --no-llm` | reports unknown project or blocked/no actionable problem, does not mutate source, does not panic, evidence behavior stays honest | generated by proof helper or test fixture |

## Safety Notes

- no `git push`
- no `git merge`
- no PR creation
- no daemon/systemd mutation
- no campaign/self-evolution
- invalid target paths are blocked before evidence/detection/provider logic
- evidence is target-local under `<target>/.eva/fix/<fix_id>/`
- arbitrary workflow paths remain forbidden
- only `.github/workflows/rust-ci.yml` is allowed in Phase 21

## How To Generate

Run the proof helper:
